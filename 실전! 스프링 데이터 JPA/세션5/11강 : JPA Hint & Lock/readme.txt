### <요약>
- QueryHints
    - 읽기 전용으로 만들기
- Lock
    - 다른이의 접근 막기

### 1. @QueryHints
- 의미
    - 읽기 전용으로 만들기
- 장점
    - 본래 DB에서 객체를 가져올 시, 객체는 기본적으로 2개가 만들어짐(원본, 수정본)
        - 객체는 1개인데 객체를 2개를 만드니 메모리 차지
    - ⇒ QueryHints로 읽기 전용을 만들면 수정이 필요없으니 객체를 1개만 만듦
- 예시(QueryHint.java)


### 2. @Lock
- 의미
    - 다른이가 접근 못하게 DB 행을 막음
- 예시(Lock.java)
    - findLockByUsername에 해당하는 테이블의 행들은 다른이가 쓰기 행위를 하지 못함
