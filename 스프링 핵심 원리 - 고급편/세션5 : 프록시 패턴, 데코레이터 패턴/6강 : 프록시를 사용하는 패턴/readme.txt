### <요약>
- 프록시를 사용하는 패턴에 대해서 소개
- 프록시
    - 사용자와 서버 사이에서 접근 제어 및 부가 기능을 추가한다.
    - 기능
        - 접근 제어
            - 권한이 없는 사용자 차단
            - 캐싱
                - (프록시가 사용자의 요청 정보를 가지고 있다면, 서버 사용안하고 직접 사용자에게 보냄)
            - 지연 로딩
                - 사용자가 요청을 하면 그때서야 프록시가 서버에게 요청
                - ex) JPA - 지연 로딩 → 프록시 객체
        - 부가 기능 추가
            - 사용자가 요청한 내용보다 더 많은 서비스 추가
            - ex) 로그 추가
- **프록시를 사용하는 패턴**
    - 프록시 패턴
        - 프록시의 접근 제어 기능이 목적
    - 데코레이터 패턴
        - 프록시의 부가 기능 추가 기능이 목적
- 호출
    - 직접호출
        - 사용자가 서버에게 요청
    - 간접호출
        - 사용자가 서버에게 요청을 보냈으나, 중간의 다른 객체가 받음
