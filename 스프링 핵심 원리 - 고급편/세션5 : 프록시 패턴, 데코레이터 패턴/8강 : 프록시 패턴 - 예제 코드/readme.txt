### <요약>
- 프록시 패턴을 구현한다
    - 프록시 패턴은 사용자 - 서버 사이에 프록시를 구현한다
    - 해당 프록시는 접근 제어(캐시) 기능이 주 목적이다.

1. 사용자 요청(ProxyPatternClient)

2. 프록시(CacheProxy)
- 사용자의 요청이 발생하면 서버에서 값 요청
- 서버에서 만들어진 값을 멤버 변수(cacheValue)에 저장
- 다시 사용자 요청이 발생하면 서버를 사용하지 않고 멤버 변수(cacheValue) 값 전달

3. 서버(RealSubject)

4. Test
- 처음에는 사용자 요청에 따라 서버 호출
- 그 이후 호출에는 프록시에 해당 값을 담기 때문에 서버 호출X
- 프록시에 있는 값 호출
