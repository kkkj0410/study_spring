### 프록시?

- 설명
- 진짜 객체를 상속받아 만든 가짜 객체
- DB를 조회하지 않고 진짜 객체를 따라해서 객체를 만듦
- 프록시는 조회한 부분에 대해서만 진짜 객체와 똑같은 데이터를 가짐
- 프록시에 없는 데이터를 조회 시, 프록시가 진짜 객체를 조회해서 데이터를 확인함
- 프록시는 진짜 객체의 자식 클래스(= 진짜 객체와 프록시는 서로 다른 클래스)
- 이점
    - 객체의 데이터가 너무 많을 때, 필요한 부분만 꺼내오는 게 가능


## 주의점

- 프록시와 진짜 객체는 클래스 타입이 다르다.
- 1차 캐시에 해당 객체가 존재하면, 프록시는 해당 객체가 된다.
