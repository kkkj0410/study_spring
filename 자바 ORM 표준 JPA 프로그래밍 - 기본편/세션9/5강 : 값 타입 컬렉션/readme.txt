### 값 타입 컬렉션?

- 값 타입(int, String 등)으로 된 List, Set 등
- 특징
    - Entity안에 삽입하여 사용
    - 해당 Entity가 만들어질 때, 값 타입 컬렉션에 해당하는 테이블 또한 만들어짐

값 타입 컬렉션의 문제점
- addressHistory는 ADDRESS 테이블의 모든 값을 삭제하고 있음
- (이후에는 수정된 값을 포함한 모든 데이터를 다시 추가함)
- ⇒ 비효율적이기 때문에 값 타입 컬렉션은 쓰지 않는다

### 해결법
- 값 타입 컬렉션을 쓰지 않는다
- 독립적으로 테이블을 만들어서 Entity와 매칭하여 사용
